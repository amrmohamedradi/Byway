# Byway LMS API — API_CONTRACT

This file is a literal, code-driven API contract for the Byway (Coursera) backend. It was generated by reading the actual controllers, DTOs, enums, exception types and the global exception middleware. Use this file as the single source of truth for frontend integration.

Important notes (literal)
- Successful responses use the `ApiResponse<T>` envelope implemented in `Coursera.Application.Common.Models.ApiResponse<T>`.
  - Shape (exact JSON):
    ```json
    {
      "success": true,
      "message": null,
      "data": { ... }
    }
    ```
- Global error handling middleware (`Coursera.Api.Middlewares.ExceptionMiddleware`) serializes exceptions as a JSON object with a single property `message` and sets status code based on exception type.
  - Error shape (exact JSON):
    ```json
    { "message": "<error message text>" }
    ```
  - Exception -> HTTP status mapping (exact logic):
    - `ValidationException` (type: `Coursera.Application.Common.Exceptions.ValidationException`) => 400 Bad Request
    - `UnauthorizedException` (type: `Coursera.Application.Common.Exceptions.UnauthorizedException`) => 401 Unauthorized
    - `NotFoundException` (type: `Coursera.Application.Common.Exceptions.NotFoundException`) => 404 Not Found
    - Any other exception type (including `System.UnauthorizedAccessException`) => 500 Internal Server Error
- Because controllers sometimes throw `System.UnauthorizedAccessException` (see Cart / Order controllers when claim not found), those cases will produce 500 with JSON `{"message":"UserId not found in token"}` unless the controller or pipeline maps it otherwise.
- Enums are used in request/response types. Read the Enums section for exact values and how to serialize them (literal behavior is stated there).

- How to interpret in responses: course DTO `CourseDto` has `Level` typed as `Level`; the JSON produced by default will be the numeric value (e.g., `2`).

### `JobTitle` (in `Coursera.Domain.Enums.JobTitle`)

```csharp
public enum JobTitle
{
    FullStackDeveloper = 1,
    BackEndDeveloper,
    FrontEndDeveloper,
    UXUIDesigner
}
```

- Integer values / names (exact):
  - `1` — `FullStackDeveloper`
  - `2` — `BackEndDeveloper`
  - `3` — `FrontEndDeveloper`
  - `4` — `UXUIDesigner`
- How it is used by the API:
  - `InstructorDto` response contains `JobTitle` as an enum typed property. The code's DTO `InstructorDto` uses `JobTitle` type; responses will serialize the enum as an integer by default.
  - When creating/updating an instructor, the controller expects the client to send `jobTitle` as a string (see `CreateInstructorRequest.JobTitle` and `UpdateInstructorRequest.JobTitle` are both `string` types). The controller then calls `Enum.Parse<JobTitle>(request.JobTitle)` — i.e., it expects the enum *name* (e.g., `"BackEndDeveloper"`) in the request body.
- Summary: responses (Instructor DTO) will have numeric enum values; requests for creating/updating instructors require the job title string name.

### Category

## SERIALIZATION/CONSUMPTION RULES (literal behavior found in code)

- Enums:
  - `Level` and `JobTitle` are defined as enums. By default the API uses System.Text.Json default settings, which serialize enums as integers. Therefore expect numeric values in responses for enum-typed DTO properties.
  - For creating/updating instructors, the controller expects the `jobTitle` field to be a string containing the enum *name* (e.g., `"BackEndDeveloper"`) because the request DTO uses `string` and the controller calls `Enum.Parse<JobTitle>(request.JobTitle)`.
- Auth:
  - Obtain `token` and `refreshToken` from `/api/auth/login` or `/api/auth/register`.
  - For protected endpoints include header: `Authorization: Bearer <token>`
- CORS: the project defines a permissive policy named `AllowAll` in `Program.cs`, but `app.UseCors("AllowAll")` is not present in the running pipeline. Until `UseCors` is added, browsers may be blocked depending on host configuration.

## FAQ / Integration notes (literal)

- How to detect a validation error vs. domain error vs. auth error?
  - Validation errors thrown using the project's `ValidationException` appear with status `400` and body `{ "message": "<text>" }`.
  - Not found situations throw `NotFoundException` and return `404` with `{ "message": "<text>" }`.
  - Authentication failures thrown using `UnauthorizedException` (project custom type) return `401` with `{ "message": "<text>" }`.
  - If a controller throws `System.UnauthorizedAccessException` (this project does in claim extraction), it will be treated as an "other" exception and return `500` with `{ "message": "<text>" }`.

- Should frontend rely on HTTP 403 for non-admin calls? Yes — ASP.NET authorization will return 403 when the token is valid but lacks required role. The middleware does not override framework authorization responses.

- Use the literal route names and parameter names exactly as shown above. Some route names are capitalized (e.g., `/api/order/Seccess`) — use the exact path.
